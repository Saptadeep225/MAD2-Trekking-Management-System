TREKKING MANAGEMENT APPLICATION - SYSTEM MANUAL & ARCHITECTURE DOCUMENT
========================================================================

1. INTRODUCTION & PROJECT ARCHITECTURE
--------------------------------------
This application is designed as a decoupled modern web application consisting of two primary halves:
- BackendFlask: A Python Flask-based RESTful API that handles data models, route logic, role-based authorization, Celery background tasks, and caching.
- FrontendVue: A Vue 3 Single Page Application (SPA) powered by Vite, compiled with Vue Single File Components (SFCs), and containing a Progressive Web App (PWA) layout for offline asset caching and homescreen installation.

Database: SQLite is used as the database management system. All tables are created automatically and seeded using an initialization script.
Caching: Flask-Caching is utilized with Redis. If Redis is offline, the backend seamlessly falls back to local memory-based caching (SimpleCache) without crashing.
Workers: Celery is hooked up to manage background services (sending emails, daily reminders, and exporting user booking histories).

Roles:
- Admin: Accesses the administrative panel, manages treks, blacklists/restores accounts, approves guides, assigns guides to routes, and views Chart.js analytics.
- Staff (Guides): Manages assigned route slots and sets route statuses (Open, Closed, Completed).
- User (Trekkers): Searches and books available treks, updates their profile, downloads booking histories as CSV files, and receives alerts.

3. HOW TO RUN THE APPLICATION
-----------------------------

A. PREREQUISITES
- Ensure python (version 3.9+) is installed.
- Ensure Node.js (version 18+) is installed.
- Ensure Redis server is installed and running locally on port 6379 (for Celery and caching).

B. BACKEND SETUP (BackendFlask)
1. Open a terminal and navigate to the "BackendFlask" folder.
2. Initialize a Python virtual environment (if not already done):
   python -m venv .venv
3. Activate the virtual environment:
   - Windows PowerShell: ..\.venv\Scripts\Activate.ps1
   - Windows Command Prompt: ..\.venv\Scripts\activate.bat
4. Install all dependencies:
   python -m pip install -r requirements.txt
5. Run the database setup script to generate tables and seed initial mock data:
   python application/init_db.py
6. Run the Flask application:
   python app.py
   - The REST API will start running on http://127.0.0.1:5000.

C. FRONTEND SETUP (FrontendVue)
1. Open a new terminal and navigate to the "FrontendVue" folder.
2. Install npm packages:
   npm.cmd install
3. Start the Vite development server:
   npm.cmd run dev
   - The Vue application will start running on http://localhost:5173/.
   - Open http://localhost:5173/ in your browser.

D. CELERY WORKER SETUP (Optional for asynchronous tasks)
1. With the backend virtual environment active, navigate to the "BackendFlask" folder.
2. Launch the celery worker process:
   celery -A application.tasks.celery_app worker --loglevel=info
3. Launch the celery beat scheduler process (in a separate terminal) to handle recurring reports:
   celery -A application.tasks.celery_app beat --loglevel=info


4. CODE STRUCTURE AND FUNCTION DETAILS

A. BACKEND FUNCTIONALITY (BackendFlask)
- app.py: Sets up the main Flask application instance. It mounts the blueprints (auth, admin, staff, user, api), enables CORS (Cross-Origin Resource Sharing) using Flask-CORS to let the Vue app call the Flask server, and registers Flask-Caching.
- application/config.py: Consolidates development settings. Configures SQLite file targets, JWT secret keys, and default Redis broker endpoints.
- application/models.py: Uses SQLAlchemy to create three tables:
  1. User: Holds id, username, email, name, phone, password (hashed), role (admin, staff, user), and status (approved, pending, blacklisted).
  2. Trek: Holds routes details, max slots, available slots, start/end dates, assigned guide ID, status, and description.
  3. Booking: Links User and Trek records, tracking registration dates and booking statuses.
- application/routes/auth.py: Houses auth services. Supports password-hashed user creation and verification, token distribution via flask-jwt-extended, and session terminations.
- application/routes/admin.py: Protects administrative resources using role checks:
  - Fetches dashboard statistics (user counts, booking totals, guide counts).
  - Queries top 5 popular treks based on reservation counts and compiles status breakdowns.
  - Controls blacklist toggle triggers for both trekkers and guides.
- application/routes/staff.py: Manages actions restricted to guides:
  - Updates remaining trek slots and sets route execution status.
  - Queries user profiles registered on coordinates assigned to that staff ID.
- application/routes/user.py: Orchestrates trekker capabilities:
  - Queries available treks based on search parameters (location, difficulty, route name).
  - Handles trek registrations (duplicate checks, slot validation, capacity decs).
  - Cancels bookings and handles CSV exports.
- application/routes/utils.py: Houses role_required decorators that validate JWT identities, blacklist statuses, and staff approval status. It also contains dictionary formatting serializers.
- application/tasks.py: Celery tasks file:
  - export_bookings_csv: Gathers trek details booked by a user, writes them to a CSV file in `static/exports/`, and writes a local notification log.
  - send_daily_reminders: Scans database for booked bookings departing tomorrow and logs mock emails.
  - send_monthly_report: Aggregates system metrics for the past month and drafts email logs.

B. FRONTEND FUNCTIONALITY (FrontendVue)
- index.html: Instantiates the single index viewport. Loads Bootstrap CSS and JS, loads Chart.js from CDN, and maps the service worker.
- public/manifest.json & public/sw.js: Configures PWA options. Caches basic CSS and font files to enable offline application load and triggers desktop installation prompts.
- src/main.js: Vue bootstrap entry point:
  - Mounts App.vue.
  - Implements Vue Router.
  - Connects a router.beforeEach guard that checks the client-side reactive store. If a page requires authorization and the user is missing a token, they are redirected to /login. If they attempt to access a console not matching their role, they are redirected to their respective role console.
- src/App.vue: Serves as the global wrapper component, displaying the navbar and router-view.
- src/store/index.js: Creates a global reactive store holding local storage bindings. If a user logs in, their token, user details, and role are cached. If they log out, the store and local storage are cleared.
- src/utils/api.js: Defines fetchAPI, a generic wrapper for API calls:
  - Automatically injects the Authorization Bearer token header if it exists.
  - Catches 401 Unauthorized responses, clears local store cache, and triggers redirect to login.
  - Provides date formatter functions and SweetAlert2 alert integrations.
- src/components/Navbar.vue: Inspects user role credentials and conditionally displays admin links, guide links, customer dashboard links, or login/register buttons.
- src/views/Home.vue: Displays introductory slides.
- src/views/Login.vue & Register.vue: Simple form controllers verifying field credentials and communicating authentication requests.
- src/views/admin/AdminDashboard.vue: Integrates Chart.js with canvas DOM elements:
  - On mount, fetches the aggregated analytics JSON data.
  - Draws a bar chart representing reservation metrics per trek.
  - Draws a doughnut chart showing booking status ratios.
  - Includes quick buttons to access other admin sections and approves pending guides.
- src/views/admin/AdminTreks.vue, AdminTrekForm.vue, AdminAssignStaff.vue, AdminStaff.vue, AdminUsers.vue, AdminBookings.vue: Handles CRUD operations for treks, blacklisting, and staff assignments.
- src/views/staff/StaffDashboard.vue, StaffTreks.vue, StaffTrekEdit.vue, StaffParticipants.vue: Allows guides to manage route slots, set route execution status, and view hikers roster.
- src/views/user/UserDashboard.vue, UserTreks.vue, UserBookings.vue, UserProfile.vue: Enables trekkers to explore routes, search by difficulty/location, register bookings, cancel bookings, request booking exports, and view notification links.


5. CORE CONCEPTS TO UNDERSTAND FOR PROJECT DEFENSE
--------------------------------------------------
- Separation of Concerns: The frontend handles display, user events, routing, and visuals. The backend handles database transactions, credentials security, and exports.
- PWA Architecture: The app works like a desktop app. By registering sw.js, key files are cached locally on the user's browser, permitting instant loading.
- Vite API Proxying: In development, Vue runs on port 5173, and Flask on port 5000. Under `vite.config.js`, we define proxies forwarding `/api`, `/auth` etc. to `http://127.0.0.1:5000`. This completely removes CORS (Cross-Origin Resource Sharing) configuration needs during local development.
- Fallback Caching: Caching speeds up listings. In `app.py`, a ping checks if Redis is online. If offline, cache fallback shifts automatically to local SimpleCache memory.
- Celery Task Workers: For operations like generating user CSV files or scanning database schedules, the Flask main thread passes tasks to Celery via a Redis message broker, maintaining UI speed for the user.
